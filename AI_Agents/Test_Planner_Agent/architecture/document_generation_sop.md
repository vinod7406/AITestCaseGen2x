# Document Generation SOP

## Goal
Convert the Markdown output generated by the LLM into a styled, downloadable `.docx` file using `Test Plan - Template.docx` if requested, or at minimum return the raw Markdown payload for the UI.

## Process
1. Receive Markdown payload from the LLM.
2. Use Python `docx` library (or `markdown2` to HTML to `docx`) to translate the headers, bold text, and lists into a readable word document.
3. Save the document in `.tmp/` or output folder.

## Fallback
If `.docx` translation fails or is complex, save simply as `.md` file since Markdown is universally readable.
